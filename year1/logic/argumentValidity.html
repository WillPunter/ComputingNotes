Arguments and Validity in Propositional Logic

  • Argument - an argument in propositional logic is a statement of the form "\(\phi_1, \phi_2, ..., \phi_n\) therefore \(\psi\)". This is often written with the double turnstile operator: \(\phi_1, \phi_2, ..., \phi_n \models \psi\).
  • Valid argument - an argument is propositionally valid if every situation in which \(\phi_1, \phi_2, ..., \phi_n\) is true, \(\psi\) is also true.
  • Valid formula - a formula is propositionally valid if it is true in every situation. Valid propositional formulas are often called "tautologies".
  • Satisfiable formula - a formula is satisfiable if it is true in at least one situation. A formula is called a contradiction if it is not satisfiable.
  • Equivalent formulas - two formulas are equivalent if they are true in exactly the same situations.
  • Valid arguments, valid formulas, satisfiable formulas and equivalent formulas can all be written in terms of each other.
    • If \(\phi \models \psi\) is a valid argument, then \(\phi \rightarrow \psi\) is a valid formula, \(\phi \rightarrow ¬\psi) is unsatisfiable and \(\phi \rightarrow \psi \equiv \top\).
  • Implication formula - for an argument \(\phi_1, \phi_2, ..., \phi_n \models \psi\), the corresponding implication formula is \(\phi_1 \wedge \phi_2 \wedge ... \phi_n \rightarrow \psi\). If an argument is valid, then the corresponding implication formula must be equivalent to \(\top\) and vice versa.
  • Checking an argument is propositionally valid - there are four ways to check if an argument is propositionally valid:
    1. Truth tables - if the conclusion is true in every row that the premise is true then the argument is valid.
    2. Direct argument - similar to mathematical proof.
    3. Equivalences - to show that an argument is valid, you can show that its implication formla is equivalent to \(\top\).
    4. Natural deduction - a way of reasoning about an argument algorithmically, using only syntactic rules. Other proof systems may be used, but we will only cover natrual deduction.
  • Checking an argument is propositionally valid using truth tables - we check if the concusion is true in every situation in which all of the premises are true.
    • E.g. the argument \(p \rightarrow ¬q, q \models ¬p\). We get the following truth table:
      \(p\)\(q\)\(¬q\)\(p \rightarrow ¬q\)\((p \rightarrow ¬q) \wedge q\)\(¬p\)
      ttttffffffff
      ttffttttffff
      ffttfftttttt
      ffffttttfftt

      We can see that whenever the premises are both true (the third row) the concusion is true, so the argument is valid.
    • Alternatively, we can create the truth table for the implication formula. If the column representing the implification formula is true on every row (i.e. in every possible situation) then the formula is equivalent to \(\top\) and the implication formula and the argument are both vaid.
  • Pros of truth tables:
    • Truth tables always work for propositional logic.
    • They can be used to show satisfiability and equivalence as well as validity.
    • They are easy to implement.
  • Cons of truth tables:
    • They are tedious and error-prone. They take a long time to generate.
    • No efficient agloritm for determining satisfiability is known, and truth-table based algorithms are particularly inefficient.
